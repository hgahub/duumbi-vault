# Release v0.4.0-preview (TUI-first)

## Context

DUUMBI core works end-to-end (v0.3.3, `init → build → run`, TUI REPL), but there is **no git tag, no GitHub Release, no prebuilt binary**, and `examples/` is empty. Issue #687 already tracks "tag v0.4.0-preview with prebuilt binaries and a working install path". The TUI is the first product surface we want to ship publicly.

## Goal

A stranger on macOS, Linux, or Windows installs DUUMBI with one documented command and reaches a working `init → build → run` plus the interactive TUI in under 10 minutes.

## Subtasks

1. Decide install channel: GitHub Releases binaries + install script as primary; crates.io publish optional (see [[2026-06-12 - Docs Truth Reconciliation]]).
2. Release CI workflow in `hgahub/duumbi`: build matrix (macOS arm64/x64, Linux x64, Windows x64 — Phase 16 already proved the MSVC target), produce signed/checksummed archives.
3. Tag `v0.4.0-preview`, create GitHub Release with changelog generated from phase history.
4. Install script (`curl | sh` style for unix, instructions for Windows) and `duumbi --version` smoke test.
5. Clean-machine install evidence on all three platforms (record as Stage-12-style evidence, like DUUMBI-382 conventions).
6. Versioning policy doc: what `-preview` means, what graduates it to v0.4.0 stable.

## Acceptance criteria

- `gh release view v0.4.0-preview` shows binaries for 3 platforms with checksums.
- Documented install path works on a clean VM for each platform.
- README and docs.duumbi.dev quickstart match the released artifact exactly.
