# Inbox-to-Atlas Processing Workflow

## Summary

Inbox-to-Atlas processing turns raw ideas, Slack captures, articles, decisions, and research requests into maintained DUUMBI knowledge notes.

## Why it matters

Raw capture is useful only if it is triaged. Without a processing workflow, the vault becomes a dumping ground and agents learn from incomplete context.

## DUUMBI usage

1. Read the Inbox item and identify whether it is execution work, durable knowledge, or both.
2. Move execution work to GitHub Project or link the existing issue/PR.
3. Create or update a Dot for atomic knowledge.
4. Update a Map when a new Dot changes navigation.
5. Update the PRD, Glossary, skill, or `AGENTS.md` only when durable guidance changes.
6. Add sources and preserve open questions.
