# Semantic Fixed Point

## Summary

A semantic fixed point is the state where user intent, graph representation, implementation, runtime behavior, and review evidence agree.

## Why it matters

DUUMBI should not treat generated code as sufficient. The value is in making behavior inspectable and consistent across all representations.

## DUUMBI usage

- Use specs to define intended behavior.
- Use graph identity to connect behavior to nodes and edges.
- Use tests and runtime evidence to validate behavior.
- Use Obsidian to preserve stable knowledge about why the behavior exists.
