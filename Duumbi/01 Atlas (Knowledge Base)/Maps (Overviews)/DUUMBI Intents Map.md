# DUUMBI Intents Map

Navigation hub for all documented DUUMBI intents.

## Purpose

Track intents generated by `/intent create`, record the audited reference version, and compare output quality across models and providers.

## How to read an intent document

Each intent Work contains:
- **Creation metadata** — command used, model, provider, timestamp
- **Acceptance criteria** — what the intent requires
- **Test cases** — concrete input/output pairs
- **Raw YAML** — the generated file as-is
- **Audited version** — manually approved reference (may differ from generated)
- **Model quality comparison** — evaluation of the same prompt across different models
