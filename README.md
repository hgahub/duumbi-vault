# DUUMBI Vault

This repository is the Obsidian knowledge vault for DUUMBI. For vault structure, onboarding order, and detailed usage rules, start with [Duumbi/How to use.md](<Duumbi/How to use.md>).

DUUMBI is an intent-driven development system where product intent, semantic structure, executable behavior, runtime feedback, and agentic workflows are connected through durable knowledge.

## Source Of Truth Rules

- GitHub Issues, PRs, CI, and Projects track execution state.
- Obsidian stores durable product, architecture, workflow, and research knowledge.
- Slack is a capture, approval, status, and follow-up surface only.
- Agent skills and `AGENTS.md` define repeatable agent behavior and repo-local rules.

## Working Rules

Keep active notes concise, linked, and source-backed. Do not duplicate live execution status from GitHub into the vault. Promote raw Inbox material into Dots, Maps, Works, GitHub issues, or skills only after triage.
